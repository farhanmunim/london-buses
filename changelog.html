v2.5 Apr 24, 2026
A new Network Overview, unified operator & garage drawers, and a full design refresh

Full design overhaul across every surface: Geist / Geist Mono typography, tokenised CSS for fonts, type scale, spacing, radii and shadows, harmonised section labels, and a matching badge language from filter pills → route-card contract badges → roadmap stages.

The right-hand panel is now a proper Network Overview. Four KPI cards (Routes, Operators, Garages, PVR) sit above a per-operator breakdown with Routes %, PVR % and EV % — tap any row and the drawer slides in with routes, PVR, a PVR-weighted Fleet Mix stacked bar and the operator's garages. Each garage opens its own drawer with the same shape plus a View all routes operated here CTA that loads those routes onto the map and stacks route cards in the Routes tab.

New in the map toolbar: Route lines, Garages, and a context-aware Stops toggle that only appears while a route is focused. Garage markers were redrawn to the original rounded-square chip design; stop popups dropped the raw NaPTAN id; route search now works identically in the topbar and the Routes tab (space / comma / enter commits a pill, Backspace drops the last one, ↑/↓/Enter on the autocomplete list).

Selection is unified across the app. A single topbar Clear all wipes every active filter, marker selection, stop filter and route pill in one click and returns to the Overview tab. A "Selected garage" pill in the sidebar mirrors the bus-stop pattern so route context is always visible. About and Roadmap modals now share a frame, width, header treatment and support-CTA, and the Changelog page runs in the same shell with a sticky topbar, fixed footer, and flat-white light-mode surface.

XLSX export picked up a Fleet Mix block in the Network overview sheet so the spreadsheet matches what you see on screen — PVR-weighted propulsion split for the currently visible routes.

Post-launch polish: a direction toggle on the route card (single-route mode, icon button next to the destination that spins 180° and flips origin → destination while the map re-renders in the new direction's colour); route-card KPIs switched from Length to Stops and now read Type · PVR · Stops · Freq · Deck · Propulsion; frequency labels are Low / Regular / High everywhere instead of raw headway bands; operator colours unified across the map, garage pins, filter pills, operator cards, drawer swatches, overview table and route-card pills; the operator drawer picked up a Garages KPI tile and both drawers share % of network routes; search-pill overflow collapses to +N so the input never gets crowded; and the mobile map HUD stacks vertically so Route lines / Garages / Stops don't fight for horizontal space on a phone.

v2.4 Apr 23, 2026
Bus-stop filter and a stops-first refresh

New in the filters panel: a Bus stop search, right below Propulsion. Pick a stop and the visible route set narrows to only the routes serving it — composed with every other filter (operator, route type, frequency, deck, propulsion) as AND. So "Go-Ahead electric routes passing through Camden Town" is one click away.

Under the hood, stops are now pre-baked weekly instead of fetched live from TfL. A new pipeline step produces data/stops.json (every London bus stop, with a reverse stop → routes index) and data/route_stops.json (per-route ordered stop lists with the "Towards …" label). Opening a route is instant, and the app behaves the same whether TfL is reachable or not.

Also: CI Actions bumped to v5 (Node 24) to clear the Node 20 deprecation warning, and the refresh pipeline grew from 8 to 9 steps to fit the new stops fetch.

v2.2 Apr 17, 2026
Accurate data: TfL-first pipeline with scrape fallbacks

Every route's operator, garage, vehicle type, PVR, deck, propulsion and destinations are now verified against the TfL Unified API, correcting a number of routes that were previously mis-assigned.

The data pipeline was rewritten to put TfL first and use londonbusroutes.net and bustimes.org only as fallbacks for the handful of routes TfL omits. TfL values are never overwritten by scraped ones.

New under the hood: numeric frequencies per peak band, a garages geodataset (postcodes.io geocoded), all London bus stops, named bus/coach stations, and a flat route_summary.csv.

v2.0 Apr 15, 2026
A completely new UI, operator garages, and multi-sheet export

London Buses has been rebuilt from the ground up. The app now runs on a proper dashboard shell with a dedicated filters panel on the left, a route detail inspector on the right, both collapsible, plus a refreshed light/dark theme.

New this release: operator garages on the map. Every London bus garage is now plotted at its real address, colour-coded by the operator's brand livery (Arriva turquoise, Stagecoach navy, Go-Ahead red, and so on). Hover for address, operator, and how many routes run from there.

Filters are now split by concern. Route type / frequency / deck / propulsion / operator drive the route lines. A separate Garage operator filter drives the garage markers independently — so you can see one operator's route network overlaid with a competitor's garage footprint.

Palette switcher. The new Colour routes by control at the top of the filters panel flips every route line between the categorical palette (type) and the operator's brand livery (Arriva turquoise, Stagecoach navy, Go-Ahead red, …) — useful for comparing network shape vs operator share on the same map.

Context-aware layer toggles in the topbar let you show / hide routes and garages globally. With a route focused, the Show routes toggle controls only the faint context underlay — so you can switch between "route only" and "route in context" without losing the selection.

Smarter data pipeline. Night-route data (N128, N29, N9, …) that was previously blank now inherits from the daytime sibling when routes.htm confirms a 24-hour service. Unknown-operator routes dropped from 71 to 37 in a single refresh.

Export now produces a single .xlsx workbook with three sheets — Routes, Garages, and a Network overview — each honouring the currently applied filters.

Elsewhere: a new dialog, per-section Clear buttons inside each filter group, fluid type + a11y pass (landmarks, focus trap on modals, keyboard-operable panel headers, focus-visible rings), and a rebuilt mobile nav with off-canvas drawers sized 92vw so the filter chips actually breathe on a phone.

v1.6 Mar 03, 2026
Route classifications and frequency filters

Routes are now classified as Regular, Prefix, Night, School, or 24-hour — each rendered in its own hue on the map so the network reads at a glance. New sidebar filters let you narrow by service frequency, deck type, and propulsion (electric, hydrogen, hybrid, diesel).

Behind the scenes: a new weekly build step scrapes londonbusroutes.net for operational details (vehicle type, garage code, PVR, frequency bands) and merges them into the route dataset.

Add multiple route numbers to the search to see them together. The selected routes render as dark-outlined coloured lines, with route-number labels pinned at each end, making it easy to compare corridors that serve the same neighbourhoods.

Operator filters and network stats

Chip-style operator filters (Arriva, First, Go-Ahead, Metroline, Stagecoach, Transport UK) let you isolate any operator's slice of the network. A new operator stats table in the sidebar shows per-operator share of routes, peak vehicle requirement, and electric-fleet percentage — recomputed live as you filter.

v1.0 Sep 08, 2025
Every London bus route on one map

First public release. An interactive Leaflet map showing every TfL bus route in London, sourced from the TfL Unified API and the TfL Bus Open Data feed. Type a route number to highlight its path, view outbound / inbound endpoints, and see every stop it serves.

Launched with an automated weekly refresh pipeline so the network is always up to date, and deployed on Cloudflare Pages.
